## 今日实现

### 1. 记忆存储路径修复：从用户工作目录改为 QuanGan 全局目录

**背景/问题：** 记忆系统上线后发现一个设计缺陷：`MEMORY_BASE_DIR` 使用的是 `CWD`（`process.cwd()`），导致在不同项目目录启动小玉时，记忆文件会分散在各个项目下，而非统一保存在小玉自己的位置。这与记忆系统的设计初衷相悖——记忆是属于小玉本身的，应全局唯一。

**涉及文件：**
- `src/memory/memory-store.ts` — 新增 `MEMORY_BASE_DIR` 常量，用 `path.resolve(__dirname, '../../.memory')` 固定到 QuanGan 项目根目录
- `src/memory/index.ts` — 重新导出 `MEMORY_BASE_DIR`，供 cli/index.ts 使用
- `src/cli/index.ts` — 所有记忆操作（`getCoreMemory`、`appendLifeMemory`、`createMemoryTools`、`createMemoryToolImpls`）的路径参数从 `CWD` 改为 `MEMORY_BASE_DIR`

**技术选型：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 继续用 `CWD` + 软链或环境变量指定 | 无需改代码 | 使用复杂，用户需额外配置 |
| **`path.resolve(__dirname, '../../.memory')`（选用）** | 与 `session-store.ts` 的 `SESSIONS_DIR` 完全一致，零配置 | 无明显缺点 |

选型理由：`session-store.ts` 已用 `path.resolve(__dirname, '../../.sessions')` 实现了同类需求（会话文件固定在 QuanGan 目录），记忆模块复用同一模式，代码风格统一且不需要用户任何配置。

**实现要点：**
- `__dirname` 在 ts-node 下为源文件所在目录，`src/memory/` 向上两级正好是项目根，因此 `path.resolve(__dirname, '../../.memory')` 得到 `<QuanGan根>/.memory/`
- `MEMORY_BASE_DIR` 作为模块级常量导出，调用方不需要自己计算路径，避免重复
