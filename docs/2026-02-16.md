# 2026-02-16 开发日志

## 今日实现

### 1. Agent 两层记忆系统（coreMemory + lifeMemory）

**背景/问题：** 小玉每次会话结束后所有上下文清空，无法积累对用户偏好和项目背景的认知。需要一套类人类记忆模型：短期用完即走，重复的事情才沉淀为长期记忆。

**涉及文件：**
- `src/memory/memory-store.ts` — 新建；封装所有文件 I/O：`getCoreMemory`/`saveCoreMemory`（core-memory.json 读写）、`appendLifeMemory`（生成带主题+日期+shortid 的 lifeMemory 文件）、`getRecentLifeMemories`（按日期过滤最近 N 天文件）
- `src/memory/tools.ts` — 新建；3 个工具定义 + 实现：`recall_memory`（关键词匹配核心记忆 + 返回最近日记列表）、`update_life_memory`（写入今日 lifeMemory 文件）、`consolidate_core_memory`（LLM 分析 14 天日记，归纳重复主题更新 coreMemory）
- `src/memory/index.ts` — 新建；导出 `createMemoryTools(client, cwd)` 工厂函数
- `src/agent/agent.ts` — `onCompressStart` 类型从 `() => void` 改为 `() => void | Promise<void>`，调用处加 `await`，支持异步回调
- `src/cli/index.ts` — 启动时读取 coreMemory 注入系统提示词；新增 `updateLifeMemoryAsync()` 函数在每次压缩时自动摘要当前会话并写入 lifeMemory，每 3 次压缩触发一次 `consolidate_core_memory`；注册 3 个记忆工具到主 Agent

**技术选型：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 结构化 KV（全量注入） | 实现简单，10 行代码 | 记忆量大时 Token 爆炸 |
| RAG 向量检索 | 记忆无限，按需检索 | 需要 embedding API，实现复杂 |
| **LLM 自动归纳 + 关键词检索（选用）** | 无外部依赖，自然语言检索，晋升机制类人类 | 检索精度受限于关键词覆盖度 |

选型理由：当前阶段无需引入向量数据库，用 LLM 自身的理解能力做记忆归纳，`reinforceCount` 字段模拟重复强化机制，后续可平滑升级为向量检索。

**实现要点：**
- 存储结构：`.memory/core-memory.json`（核心记忆，单文件）+ `.memory/life/lifeMemory-<主题>-<日期>-<shortid>.md`（每日日记，多文件）
- `coreMemory` 晋升机制：每次 lifeMemory 更新后（每 3 次压缩触发一次），LLM 对比近 14 天日记与现有核心记忆，识别重复出现 2+ 次的主题，自动 `reinforceCount +1` 或新增条目
- `onCompressStart` 原来是同步 `() => void`，改为 `await this.onCompressStart()` 支持异步，lifeMemory 更新逻辑才能 fire-and-forget 不阻塞压缩
- 系统提示词动态注入：启动时把 `coreMemory.memories` 全量追加到小玉的提示词末尾，并附加 `recall_memory` 使用指南（遇到具体项目/人物/历史决定时调用，闲聊不调用）

## 关键收获

1. **两层记忆架构类人类**：lifeMemory 对应「日记/情节记忆」，coreMemory 对应「长期记忆/内隐记忆」；触发晋升的关键是重复性而非单次重要性，这与人类记忆强化机制一致
2. **回调函数的异步化**：将 `() => void` 改为 `() => void | Promise<void>` 是不破坏现有调用方的最小改动，`await callback?.()` 和 `if (cb) await cb()` 都可接受，后者在 TS 中更类型安全
3. **fire-and-forget + 静默失败**：lifeMemory 更新是辅助功能，不应阻塞主流程。用 `.catch(() => {})` 吞掉错误，确保记忆系统的任何异常都不会影响用户体验
4. **工厂函数天然解决依赖注入**：`createMemoryTools(client, cwd)` 让 LLM client 和存储路径在调用层决定，工具实现内部无全局状态，可复用可测试
